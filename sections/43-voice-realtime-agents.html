43.1 Vì sao voice agent khác text agent

Voice không chỉ là "text-to-speech của text agent". Có 5 khác biệt cốt lõi mà nếu bỏ qua sẽ làm sản phẩm voice cảm giác "khó dùng" dù model rất giỏi:

  • Latency budget khắt khe: text chat user chấp nhận đợi 2–3 giây cho first token; voice user cảm giác "lag" nếu đợi quá ~800ms. Anything >1.5s là gãy luồng đối thoại.
  • Không có "undo": chat có thể edit message; voice một khi đã nói là user nghe rồi. Hallucination thành thiệt hại trực tiếp.
  • Barge-in (interrupt): con người ngắt lời nhau liên tục. Voice agent không xử lý được interrupt sẽ cảm giác robot, "talking over me".
  • Prosody mang ngữ nghĩa: "Yeah, right" với intonation khác nhau có thể khẳng định hoặc mỉa mai. Text agent mất 100% thông tin này; voice agent cần model giữ được paralinguistic cue.
  • Streaming bắt buộc: không thể "đợi response xong rồi đọc"; TTS phải bắt đầu phát ngay khi token đầu tiên ra, parse chunk-by-chunk.
!
Voice agent là realtime system, không phải LLM app có TTS

Nếu approach của bạn là "thêm TTS vào ChatGPT-clone", trải nghiệm sẽ giống IVR của tổng đài 2005 — robot, slow, không hiểu khi user ngắt lời. Voice agent thực thụ là một realtime streaming system với end-to-end latency budget <800ms.

43.2 Hai kiến trúc: pipeline vs speech-to-speech

2026 có hai trường phái rõ rệt để build voice agent:

Tiêu chí Pipeline (STT → LLM → TTS) Speech-to-Speech (S2S native)
Latency E2E 1.5–3s (cộng dồn 3 step) 300–800ms
Prosody preservation Mất — STT chỉ trả text Giữ — model "nghe" trực tiếp
Barge-in Phải tự xử (VAD + cancel) Native support
Tool/function calling Dễ — LLM tier text, đã chín Đang dần chín (Realtime API)
Model lựa chọn Nhiều — mix & match Ít — chủ yếu OpenAI/Google
Cost Rẻ — 3 component có offering miễn phí/cheap Đắt hơn — audio token billing
Quality dialogue OK, hơi "robot" Tự nhiên hơn rõ rệt

Recommendation 5/2026: Bắt đầu với Pipeline để control + cost, chuyển sang Speech-to-Speech khi cần dialogue quality cao (customer support cao cấp, voice tutor, accessibility). Có thể hybrid: S2S cho conversational layer, fallback pipeline khi cần tool đặc thù.

43.3 Realtime API (OpenAI, Gemini Live)

Cả OpenAI và Google đều có Realtime API dùng WebSocket (hoặc WebRTC qua proxy) để streaming audio hai chiều — không cần dựng pipeline STT/TTS riêng: 

# OpenAI Realtime API — voice-to-voice với GPT-5.5 Realtime
import asyncio, base64, json
from websockets import connect

async def voice_agent():
    url = "wss://api.openai.com/v1/realtime?model=gpt-5.5-realtime"
    headers = {"Authorization": f"Bearer {API_KEY}"}
    async with connect(url, extra_headers=headers) as ws:
        # Config: voice, instructions, tools, turn_detection (VAD)
        await ws.send(json.dumps({
            "type": "session.update",
            "session": {
                "voice": "alloy",
                "instructions": "You are a friendly support agent. Speak naturally.",
                "turn_detection": {"type": "server_vad", "threshold": 0.5},
                "tools": [{"type": "function", "name": "check_order_status", ...}],
                "input_audio_transcription": {"model": "whisper-v3-turbo"},
            }
        }))
        # Stream mic audio in & speaker audio out concurrently
        await asyncio.gather(pump_mic(ws), handle_events(ws))

Vài thứ quan trọng khi dùng Realtime API:

  • Server-side VAD: model tự detect khi user dừng nói, không cần bạn implement. Threshold 0.5 default; tăng để bớt nhạy với noise.
  • Input transcription: model "hiểu" audio trực tiếp, nhưng vẫn cho phép request Whisper transcribe song song để log + analytics.
  • Tool calling: vẫn hoạt động bình thường, nhưng output đến qua event response.function_call_arguments.delta. Khi tool chậm, model tự "filler" ("let me check..."). Cài interrupt_response: false nếu không muốn user ngắt được mid-tool.
  • Pricing 2026 (audio token): input ~$32/1M token, output ~$64/1M — đắt hơn text 10–20×. Một cuộc gọi 5 phút ≈ $0.30–0.80.

Gemini Live (Google) có model tương tự, kèm vision-realtime — user có thể share camera, model nhìn + nghe + nói cùng lúc. Use case: vision-aided tutoring, accessibility ("describe what you see").

43.4 STT & TTS hiện đại

Nếu chọn pipeline (cost / flexibility), 2 component phải pick cẩn thận:

STT (Speech-to-Text)

  • Whisper v3-turbo (OpenAI, 10/2025): 6× nhanh hơn Whisper v3, quality không kém. Multi-lingual, có timestamp. Default cho self-host. Hỗ trợ tốt tiếng Việt.
  • Deepgram Nova-3: streaming-first, latency <300ms, có speaker diarization. Pricing thấp hơn Whisper Cloud cho high-volume.
  • AssemblyAI Universal-2: tốt cho noisy audio, có auto-PII-redaction, sentiment analysis built-in.
  • NVIDIA Parakeet TDT: open-source, chạy nhanh trên GPU, 600× realtime cho transcription batch.

TTS (Text-to-Speech)

  • ElevenLabs v3 / Multilingual v2.5: SOTA cho tự nhiên + cảm xúc, voice cloning từ 3s sample. Conversational AI suite (5/2025) bundle TTS + LLM proxy + telephony.
  • Sesame CSM-1B (open-source, 3/2025): Speech-to-speech native nhỏ, chạy được on-device. Tốt cho mobile app cần voice quality không gửi cloud.
  • Cartesia Sonic 2: latency 40ms first-chunk — nhanh nhất 2026, dùng cho IVR cần response <500ms tổng.
  • OpenAI gpt-4o-mini-tts: rẻ, ok quality, instruction-following ("speak slower", "in a calm tone") — voice control bằng prompt.

43.5 Latency budget & barge-in

Voice UX bị chi phối bởi end-to-end latency từ khi user dừng nói đến khi audio response phát ra. Budget tham khảo cho dialogue tự nhiên:

≡
Latency budget cho voice agent (target 2026)
  • <500ms: feel "snappy", không nhận ra delay. Hard target cho IVR.
  • 500–800ms: comfortable, default cho assistant.
  • 800ms–1.5s: noticeable lag, user bắt đầu mất kiên nhẫn.
  • >1.5s: user nghĩ "máy hỏng" hoặc tự nói thêm — broken UX.

Cách bóp latency xuống <800ms cho pipeline:

  1. VAD streaming: detect end-of-utterance tại chỗ (Silero VAD), đừng đợi 1s silence rồi mới send STT.
  2. STT streaming: dùng partial transcript ngay khi có chunk — Whisper turbo + chunk decode, hoặc Deepgram streaming.
  3. LLM streaming + early TTS: bắt đầu TTS ngay khi LLM emit sentence đầu (đợi đến dấu . hoặc ? rồi push vào TTS); đừng đợi full response.
  4. Speculative TTS (advanced): TTS pre-generate cho top-K likely next sentence, drop nếu sai. Tăng cost ~20%, giảm latency 200ms.
  5. Reasoning model: avoid extended thinking cho voice — thinking token làm TTFT lên 2–5s. Nếu task thực sự cần reasoning, dùng "filler" ("Để tôi nghĩ xíu...") để mask delay.

Barge-in handling

Khi user ngắt lời agent đang nói, hệ thống phải:

  • Detect user voice over agent audio (echo cancellation đã good).
  • Stop TTS immediately — fade-out 100ms để không "cắt giọng" lốp đốp.
  • Cancel in-flight LLM call — request mới sẽ override.
  • Update context: agent phải biết "tôi đã nói được đến đâu trước khi bị ngắt" — không phải lúc nào cũng cần (Realtime API tự handle), nhưng pipeline phải tự track.

43.6 Tool use trong voice

Voice agent thường cần tool (check booking, lookup database, submit ticket). Vài pattern riêng cho voice:

  • Always speak before tool call: user không nhìn loading spinner. Phải có "Để tôi check..." (filler) trước khi tool chạy, không thì user nghĩ máy đứng.
  • Streaming tool result: kết quả tool dài → đừng đợi hết rồi đọc. Stream từng item ("Tôi tìm thấy 3 booking. Booking đầu tiên là...").
  • Tool timeout < conversation budget: tool chậm hơn 3s phải có timeout + fallback. User không chờ 10s im lặng.
  • Confirm trước khi mutate: voice dễ nhầm — "xoá booking 1234" và "xem booking 1234" nghe rất giống. Mutating tool (book/cancel/pay) luôn cần confirm bằng giọng + đọc lại tham số.

43.7 Eval cho voice agent

Eval voice phức tạp hơn text 3× vì có 3 dimension độc lập cần đo:

DimensionMetricTool / approach
ASR accuracy WER (Word Error Rate), CER (Character Error Rate cho tiếng Á) Golden audio set + manual transcript; jiwer library
Task success Goal completion rate, turns-to-resolution LLM-as-judge trên full transcript + tool log
Voice quality MOS (Mean Opinion Score), naturalness, prosody Crowdsourced rating (Prolific), UTMOS automated
Latency P50/P95 E2E, time-to-first-audio OTel GenAI traces + custom audio timestamp
Barge-in correctness % interrupts handled gracefully Adversarial test set: deliberate interrupts
Safety Refusal accuracy, PII leak in audio Red-team audio prompts; output audio re-transcribed + scan

43.8 Pattern sản phẩm 2026

  • Voice IVR replacement (call center): pipeline + ElevenLabs + Twilio. Cost < human, quality dialogue đủ cho L1 support. Top vendor: Bland AI, Vapi, Retell.
  • In-app voice mode (mobile): S2S native qua Realtime API, hoặc on-device (Sesame CSM, Apple Foundation Models speech). Use case: ChatGPT voice, Pi, Replika.
  • Voice tutor / coach: cần prosody chuẩn (correct pronunciation), ElevenLabs + custom dictionary. Tools: Speak, Praktika.
  • Voice accessibility: pair với computer-use agent — user nói lệnh, agent thực thi trên UI. Apple Voice Control + LLM, Microsoft Copilot Voice Action.
  • Outbound voice agent (sales/scheduling): gọi điện thay người. Legal sensitive — phải có disclosure "I'm an AI" và tuân FCC TCPA rules.
!
Voice cloning & deepfake — handle với care

ElevenLabs v3 clone giọng từ 3 giây sample. Đa số platform có "voice consent" verification, nhưng workaround vẫn nhiều. Nếu sản phẩm cho phép clone, bắt buộc: consent recording lưu lại 5 năm, watermark audio output (ElevenLabs có Inaudible Audio Watermark), và disclosure rõ ràng. EU AI Act Article 50 đã require disclosure cho generated audio từ 2026.

Tóm tắt

  1. Voice agent ≠ "text agent + TTS". Khác biệt cốt lõi: latency budget <800ms, barge-in, prosody, streaming bắt buộc, không có undo.
  2. 2 kiến trúc: Pipeline (STT → LLM → TTS, control + cost) vs S2S native (Realtime API, latency + dialogue quality). 2026 default: Pipeline cho enterprise, S2S cho consumer app.
  3. Realtime API (OpenAI, Gemini Live) cung cấp WebSocket S2S với server-side VAD, native tool calling, audio token billing 10–20× text.
  4. STT pick: Whisper v3-turbo (default), Deepgram Nova-3 (streaming), AssemblyAI (noisy). TTS: ElevenLabs (quality), Cartesia (latency), Sesame CSM (on-device).
  5. Latency budget <800ms cần stream end-to-end: streaming VAD + STT partial + LLM token streaming + early TTS từ sentence đầu.
  6. Tool use riêng cho voice: filler trước call, stream result, confirm trước mutate, hard timeout.
  7. Eval phải đo 6 dimension độc lập: ASR (WER), task success, voice quality (MOS), latency (P95), barge-in handling, safety (PII leak in audio).
  8. Pattern 2026: IVR replacement (call center), in-app voice (mobile S2S), voice tutor, voice accessibility, outbound (legal-sensitive).
  9. Voice cloning có risk lớn — luôn kèm consent + watermark + disclosure; EU AI Act Article 50 require từ 2026.
